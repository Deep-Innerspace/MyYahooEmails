== YOUR TASK ==

I will upload an Excel workbook. It contains 2 sheets:

--- Sheet 1 — "Instructions"
Background information. Read it for context. Do not modify it.

--- Sheet 2 — "Emails" (INPUT + OUTPUT)
Each row is one email. Blue columns = read-only input. Yellow columns = fill these.

Input columns (blue — DO NOT MODIFY):
  email_id   : Unique numeric ID
  date       : Email date (YYYY-MM-DD)
  direction  : "sent" (from father) or "received" (from mother)
  subject    : Email subject

  event_date   : Date of the event in YYYY-MM-DD format (or YYYY-MM or YYYY if only partial)
  event_type   : One of the types listed below
  significance : "high", "medium", or "low"
  description  : One sentence in French describing the event precisely


== EVENT TYPES ==

Use exactly one of these values in the event_type column:

  statement      : A factual claim about what happened ("j'ai payé le loyer le 3 mars")
  commitment     : A promise or agreement made ("je te rendrai les enfants dimanche")
  accusation     : An allegation against the other party ("tu n'as pas payé la pension")
  denial         : A denial of an allegation ("je n'ai jamais dit cela")
  legal_fact     : Reference to a court order, judgment, or legal proceeding
  financial_fact : A monetary fact, payment, debt, or asset value
  child_fact     : A fact about the children (school, health, custody, activities)
  property_fact  : A fact about the shared apartment, assets, or indivision
  demand         : An explicit written request or demand


== SIGNIFICANCE ==

  high   : Legally significant — contradicts other claims, involves children's welfare,
           references a court order, or documents a broken commitment. Usable as evidence.
  medium : Notable fact — documents an agreement, payment, or key event that establishes context.
  low    : Routine — logistical information, minor schedule detail, or passing reference.


== RULES ==

1. ONE EVENT PER ROW — extract only the MOST significant event from each email.
   If the email contains several events, pick the one with the highest legal relevance.

   (e.g. pure logistics, greetings, acknowledgements with no facts, too vague).
   Do NOT invent events. Do NOT fill in rows where there is nothing to extract.

3. event_date is the date the EVENT occurred or was referenced — NOT the date of the email.
   If the email says "j'ai payé le 15 mars", the event_date is the 15th, not the email date.
   If only a month/year is mentioned, use YYYY-MM. If only a year, use YYYY.
   If the date is approximate or inferred from context, still fill it — use your best estimate.

4. Descriptions must be in French, factual, and precise.
   Quote verbatim key words or amounts from the email where possible.
   Max 2 sentences.

5. Do NOT modify the blue input columns.
